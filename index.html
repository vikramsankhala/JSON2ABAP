Application Logic

  1. Parse JSON – Validates and parses the input JSON string.
  2. PascalCase → snake_case – Converts JSON keys (e.g. TranDtls, ItemList) to ABAP field names (tran_dtls, item_list) to match xco_cp_json=>transformation->pascal_case_to_underscore.
  3. Type inference – Maps JSON types to ABAP: string→string, number (integer)→i, number (decimal)→decfloat16, boolean→abap_bool, object→nested structure, array→STANDARD TABLE OF ... WITH DEFAULT KEY.
  4. Nested structures – Recursively generates TYPES: BEGIN OF ty_xxx ... END OF ty_xxx for objects; defines child types before parent for correct dependency order.
  5. Deserialize call – Outputs the xco_cp_json=>data->from_string()->apply()->write_to() pattern with PascalCase and boolean transformations.
